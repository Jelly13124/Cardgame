# Asset Spec — UI Overhaul (Slice 1A → 1D)

**Audience:** Codex (asset generation pipeline)
**Owner of code/JSON:** Claude (already implementing)
**Project:** Cardgame (Godot 4.6, Hardcore 128 Pixel Wasteland)
**Active art ADR:** [0008](adr/0008-art-pivot-to-hardcore-128-pixel-wasteland.md)
**Palette ADR:** [0009](adr/0010-third-palette-recalibration.md)

This is the **comprehensive UI asset contract** for the four-slice UI overhaul (1A spec → 1B card render → 1C battle HUD → 1D scene polish). Codex can start delivering items in any order — each component is independently usable.

If an asset is missing, the game falls back to placeholder visuals (defined in code) or shows a "missing texture" warning. Nothing crashes — but it looks bad.

## 1. Pipeline Reminders

Per `docs/project-rules.md` §2-§5:

- **Native resolution:** 128 px target for all hand-painted UI components (panels, frames). Small icons (status badges, intent icons) can be 64 px native if 128 wastes space.
- **Background:** transparent PNG. Sheet-generation magenta (`#FF00FF`) is OK during generation; final output must be transparent.
- **`.import` files** are generated by Godot on first import — do not write manually.
- **Pipeline intermediates** (`raw-*.png`, `*-clean.png`, `sheet-transparent.png`) belong in a `generated_sheet/` subfolder, NOT alongside final assets.
- **9-slice components** (panels, buttons) MUST have art-safe margins so the center can stretch without distortion. Recommended: 16 px corner margins for 128 px components, 8 px for 64 px.

Categorized by use. Each component lists: output path, dimensions, variants, consumer code, prompt hint.

### 2.1 Panels (2 components, 9-slice)

#### `panel_default.png` — generic panel background
- **Path:** `battle_scene/assets/images/ui/panel_default.png`
- **Native:** 128×128 px, 9-slice with 16 px corner margins
- **Visual:** Rusted metal plate with bolt rivets in the corners, dark `PANEL_BG` interior. Bold `LEATHER_DARK` outline. Slight ware/scratches in the metal — should tile invisibly.
- **Used by:** Loot reward panels, settings menus, generic info dialogs
- **Prompt hint:** Rusted iron sheet panel with rivets in corners, scratched metal texture, dark interior, transparent 16px border safe-zone, 9-slice tileable center, hardcore wasteland UI element

#### `panel_dark.png` — modal backdrop / inspect overlay
- **Path:** `battle_scene/assets/images/ui/panel_dark.png`
- **Native:** 128×128 px, 9-slice with 16 px corner margins
- **Visual:** Much darker than `panel_default` — `PANEL_BG_DARK` interior. Heavier border. Used when something needs to feel "modal" (covering gameplay below).
- **Used by:** Relic choice modal, card inspect overlay, settings menu
- **Prompt hint:** Heavy dark armored panel, riveted black-brown metal, thick borders, deep shadow interior, 9-slice tileable, oppressive heavy UI panel

### 2.2 Buttons (3 components, 9-slice)

#### `button_normal.png` / `button_hover.png` / `button_pressed.png`
- **Path:** `battle_scene/assets/images/ui/button_normal.png` (and `_hover.png`, `_pressed.png`)
- **Native:** 96×48 px each, 9-slice with 12 px horizontal / 8 px vertical corner margins
- **Visual:**
  - `normal` — leather-strap or stamped-metal button with RUST_PRIMARY warm tone, LEATHER_DARK border
  - `hover` — same but with ACCENT_NEON_BLUE edge glow / brighter face

- **Used by:** Every Button node project-wide (END ROUND, Proceed, Skip Draft, Settings, Resume, Exit, etc.)
- **Prompt hint (normal):** Wasteland-style stamped metal button with leather edge strap, warm rust palette, dark border, tileable 9-slice button face, neutral state
- **Prompt hint (hover):** Same button but with subtle electric-blue edge glow and slightly brighter face
- **Prompt hint (pressed):** Same button but depressed/inset, with hot-orange edge glow indicating active press

## 5. Verification

After Codex delivers a batch, the human should:

1. Run `"/c/Program Files/Godot/godot" --headless --path . --quit-after 5` — should print `DataValidator: all card/enemy/relic JSON files passed schema check.` (sprite quality isn't validated but project must still parse).
2. Open the editor; the asset re-imports automatically.
3. Visually confirm in-game: launch a quick battle, look at HUD / cards / intent badges. Existing code is already wired to load these PNG paths — no code change needed for REPLACE assets.

NEW assets (`panel_default`, `panel_dark`, `button_*`, `status_badge_bg`, `intent_charge`) will land but NOT visually appear until **Slice 1B/1C/1D** wires them into the relevant scenes. That's intentional — Codex can deliver them now and Claude will wire them later.
