## Step 3 — Pipeline rules (non-negotiable)

These come from `docs/project-rules.md` §2–§5. Follow exactly:

1. **Every prompt** must preserve the Hardcore 128 Pixel Wasteland Art prompt anchor (see asset spec §0).
2. **Item icons are item-only** — no character holding the item, no UI frame, no background scene, no rarity badges, no text. Just the item, centered, on transparent background.
3. **Sheet generation uses `#FF00FF` (magenta) background** for chroma-key cleanup. Final per-item output must be transparent PNG.
4. **Output path is exact**: `battle_scene/assets/images/equipment/{item_id}.png` where `item_id` matches the JSON's `id` field exactly. Wrong filename = icon doesn't load (placeholder fallback).
5. **One neon accent color per item** — small detail, not flood-color the whole item.
6. **`.import` files** are generated by Godot on first import. Do not write them manually.
7. **Intermediate sheets** may stay in `battle_scene/assets/images/equipment/generated_sheet/` if they match the current delivery — drop a `pipeline-meta.json` and `prompt-used.txt` there for traceability.

## Step 4 — One item at a time

For each item in section 3 of the asset spec:
1. Compose the prompt: theme description from the asset spec entry + neon accent color + the mandatory style anchor suffix.
2. Generate the icon (with `#FF00FF` background).
3. Chroma-key cleanup → single transparent PNG.
4. Verify the silhouette reads at small size (the panel will render it in a 48×48 slot, so squint at a 48px-scaled preview to confirm legibility).
5. Save to `battle_scene/assets/images/equipment/{item_id}.png`.
